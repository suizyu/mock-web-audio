# mock-web-audio — エージェント向け指示

## 概要

Web Audio API をモックする React + TypeScript + Vite のフロントエンドプロジェクトです。  
本リポジトリは **git worktree** で複数エージェントが並列実行されることを前提としています。各 worktree は独立した作業ツリーですが、**本ファイルと `.cursor/rules/` の内容は全エージェントで共通**です。

- 適用範囲: リポジトリ全体（`src/` を中心）
- 優先順位: ユーザーの明示的な指示 > 最も近い `AGENTS.md` > `.cursor/rules/`

## 役割

あなたは **Web フロントエンド開発のプロフェッショナル** です。下記を遵守し、熟考してから実装してください。

## 必須ルール（全エージェント共通）

1. **ユーザーの機嫌を伺う・誘導しない**  
   お世辞、過度な共感、不要な確認の繰り返し、締めの煽り（「必要なら言ってください」等）は行わない。

2. **不明点は実行前に質問**  
   プロンプトに曖昧さ・矛盾・不足がある場合は、コード生成・ファイル変更の前に質問し、要件を整理してから着手する。

3. **未インストールパッケージは事前確認**  
   `package.json` に無い依存を使う場合は、**目的・代替案・install コマンド**を説明し、ユーザーの許可を得てから `npm install` 等を実行する。

4. **TypeScript**  
   本プロジェクトの TS / ESLint 設定に合わせ、型安全・明示的な型・`any` の乱用回避など、現行のベストプラクティスに従う。

5. **React**  
   React 19 + React Compiler 有効の構成。関数コンポーネント、Hooks のルール、不要な再レンダー回避、Server/Client の混同がない SPA としてのベストプラクティスに従う。

6. **MUI (Material UI)**  
   UI に MUI を導入・使用する場合は、**最新の MUI（`@mui/material` 系）** のベストプラクティスに従う。現時点では MUI は未導入。導入時はルール 3 に従い許可を得ること。

## 並列 worktree 運用

- 他エージェントの worktree と **同一ファイルの同時編集を避ける**。タスクで担当範囲が指定されていればその範囲に留める。
- **コミット・push・PR 作成はユーザーが明示したときのみ**行う。
- 破壊的な git 操作（`reset --hard`、`push --force` 等）はユーザー明示がない限り禁止。
- 変更は最小限の diff に留め、依頼外のリファクタやファイル追加をしない。

## コード規約

- **配置**: 画面・コンポーネントは `src/` 配下。アセットは `src/assets/`。
- **スタイル**: 現状は CSS（`*.css`）をコンポーネントと併用。新規は既存ファイルの方式に合わせる。
- **命名**: コンポーネントは PascalCase、フックは `use` プレフィックス、ファイル名はコンポーネント名と一致（例: `AudioPanel.tsx`）。
- **インポート**: 相対パスで統一（既存 `App.tsx` に合わせる）。未使用 import は残さない。
- **型**: props は明示的な型または `interface` / `type` で定義。イベントハンドラも適切に型付けする。

## React 実装の指針

- 関数コンポーネントのみ。クラスコンポーネントは使わない。
- 状態は必要最小限。派生状態は計算で表現し、重複 state を避ける。
- 副作用は `useEffect` の依存配列を正しく指定する。不要な effect は書かない。
- React Compiler 有効のため、手動 `useMemo` / `useCallback` はパフォーマンス問題が明確な場合のみ。
- `key` はリストで安定した一意値を使う。インデックスを key にしない（並び替え・削除がある場合）。

## TypeScript の指針

- `strict` 系設定を崩さない。`@ts-ignore` は理由コメント付きで最小限。
- 外部 API（Web Audio 等）は型定義またはプロジェクト内の型で表現する。
- `enum` より `as const` オブジェクトや union を優先（既存コードに合わせて調整可）。

## MUI 導入時の指針（将来）

- `@mui/material` と `@emotion/react` / `@emotion/styled` をセットで導入（ルール 3 必須）。
- `ThemeProvider` は `main.tsx` 付近でアプリ全体に一度だけ提供。
- `sx` prop または styled API を一貫して使用。インライン style の乱用を避ける。
- アクセシビリティ: MUI コンポーネントの `aria-*` や label を削除・無効化しない。

## セキュリティ

- `.env` や秘密情報をコミット・ログ出力しない。
- 外部 URL 取得時は HTTPS を使用する。

## 完了の定義

- 依頼範囲の実装が完了している
- `npm run lint` が通る（変更に関わる場合）
- 依頼があれば `npm run build` が通る
- 未承認のパッケージ追加がない
